# 健康检查说明 / Health Check

> 知识库会随时间长歪：资料没整理、页面缺来源、概念重复、链接断裂……
> 定期做一次健康检查，把问题找出来，结果写进本目录（如 `health-YYYY-MM-DD.md`），并在 [[wiki/log]] 记一笔。

---

## 检查清单

每次检查至少覆盖以下几项：

1. **缺来源摘要的 raw 资料**
   `raw/` 里有哪些原始文件，还没有对应的 `wiki/sources/` 摘要页。

2. **缺来源的页面**
   `wiki/` 里哪些概念页 / 实体页 / 综合页里的判断没有指向来源。

3. **重复或冲突的概念**
   哪些概念页讲的是同一件事（该合并），或互相矛盾（该核对）。

4. **没有链接的孤立页面**
   哪些页面既没有出链、也没有入链（孤岛），需要补 `[[ ]]` 关联。

5. **可能过期的判断**
   哪些结论依赖的资料已更新、或时效性强，需要重新核验。

6. **目录与标签不一致的 raw 文件**
   - `raw/articles/` 里的文件是否都打了 `article`（公众号是否补了 `wechat`）。
   - `raw/clips/` 是否都是 `clip`。
   - `raw/papers/` 是否都是 `paper`。
   - 是否还有残留的 `clippings` 旧标签（应清理）。
