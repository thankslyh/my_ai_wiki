# 维护规则 / AGENTS

> 以后任何人（或任何 AI agent）维护这个知识库，都必须遵守下面的规则。
> 目标：保证资料可追溯、不丢失、不编造、可持续整理。

---

## 1. 原始资料不可变

- `raw/` 只保存**原始资料**，**只增不改**：不删除、不覆盖、不就地编辑。
- 需要修正或补充时，新增一份文件，不动原文件。

## 2. 整理后的知识进 wiki/

- `wiki/` 存放整理后的知识条目：
  - `wiki/sources/` —— 来源摘要页
  - `wiki/concepts/` —— 概念页
  - `wiki/entities/` —— 实体页
  - `wiki/syntheses/` —— 综合页

## 3. 每个重要判断都指向来源

- 凡是结论、观点、数据，尽量标注来源（链接到对应的来源摘要页，或附原始链接）。
- 没有来源支撑的判断，不要写成既定事实。

## 4. 新资料的标准处理流程

新资料进来后，**按顺序**执行：

1. 把原始文件放进对应的 `raw/<类型>/` 目录，并打正确标签（见第 7 条）。
2. 先生成**来源摘要页**（`wiki/sources/`，套用模板 [[wiki/sources/README]]）。
3. 再更新**概念页**（`wiki/concepts/`）。
4. 再更新**实体页**（`wiki/entities/`）。
5. 再更新**综合页**（`wiki/syntheses/`）。
6. 更新 **`wiki/index.md`**（主题、入口、各类页面清单）。
7. 更新 **`wiki/log.md`**（记录本次新增与改动）。

## 5. 不确定就标注，不要编

- 拿不准的信息标注 **`待核验`**；缺失的内容标注 **`待补充`**。
- 宁可留空+标注，也不要编造事实、链接或数字。

## 6. 每次整理完，报告改动

- 整理完成后，明确列出**改了哪些文件**（新增 / 修改 / 删除）。

## 7. 目录与标签对应关系

| 目录 | 标签 |
| --- | --- |
| `raw/articles/` | `article`（微信公众号文章追加 `wechat`） |
| `raw/clips/` | `clip` |
| `raw/papers/` | `paper` |

- **不再使用 `clippings` 作为 raw 类型标签**。
- 标签写在 Markdown frontmatter 的 `tags` 字段，或正文 `#标签`，保持与所在目录一致。

## 8. 健康检查

- 定期按 [[outputs/health/README]] 的清单做健康检查，结果写进 `outputs/health/` 并在 `wiki/log.md` 记一笔。
